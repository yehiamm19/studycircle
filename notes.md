# StudyCircle Project Notes

---

# 📘 Project Overview

## Project Name
**StudyCircle**

## Project Type
Collaborative Study Management & Agile Productivity Platform

## Project Goal
StudyCircle is designed to help university students collaborate effectively, manage academic tasks professionally, and apply Software Engineering methodologies in a practical educational environment.

The platform combines:
- Agile methodologies
- Scrum frameworks
- Kanban task management
- Pomodoro productivity systems
- Gamification
- Collaborative communication
- Requirement traceability

The goal is to simulate real-world software engineering teamwork while improving student productivity and engagement.

---

# 🎯 Main Objectives

- Improve collaboration between students.
- Enhance productivity and focus during study sessions.
- Apply Agile and Scrum methodologies practically.
- Provide visual workflow management using Kanban boards.
- Encourage teamwork through role-based collaboration.
- Improve requirement traceability and project organization.
- Create an engaging educational platform using gamification.
- Simulate modern SaaS productivity systems used in industry.

---

## MoSCoW Prioritization

Each task is categorized using MoSCoW prioritization:

| Priority | Meaning |
|----------|----------|
| Must Have | Critical requirement |
| Should Have | Important but not critical |
| Could Have | Optional enhancement |
| Won’t Have | Not included in current sprint |

### Purpose:
Helps prioritize tasks professionally during sprint planning.

---

## Story Points
Tasks include story point estimation to measure effort.

### Examples:
- 2 Points → Small task
- 5 Points → Medium task
- 8 Points → Complex task

### Purpose:
Supports Agile estimation and velocity calculations.

---

## Requirement Traceability
